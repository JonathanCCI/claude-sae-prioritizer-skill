# Scoring Criteria

Score each account 1-10 on five weighted criteria.

## Formula

```
totalScore = (need × 0.30) + (current × 0.25) + (market × 0.20) + (news × 0.15) + (contacts × 0.10)
```

Focus accounts: +0.5 bonus (capped at 10)

## Criteria

### 1. CI/CD Need (30%)

| Score | Indicators |
|-------|------------|
| 9-10 | Engineering-heavy, microservices, complex pipelines, 100+ engineers |
| 7-8 | Significant dev activity, multiple teams, growing eng org |
| 5-6 | Moderate development, some automation needs |
| 3-4 | Limited software development |
| 1-2 | Minimal technical footprint |

### 2. Current Solution (25%)

| Score | Indicators |
|-------|------------|
| 9-10 | Jenkins (migration candidate), legacy CI, known pain points |
| 7-8 | Mixed tooling, consolidation opportunity |
| 5-6 | GitHub Actions at scale (pricing pressure) |
| 3-4 | Established CI/CD, unclear dissatisfaction |
| 1-2 | Already CircleCI or deeply invested competitor |

### 3. Market Timing (20%)

| Score | Indicators |
|-------|------------|
| 9-10 | GitHub Actions pricing impact, active Jenkins migration |
| 7-8 | Cloud migration underway, peers switching CI/CD |
| 5-6 | General DevOps modernization applicable |
| 3-4 | No timing pressure identified |
| 1-2 | Recently completed CI/CD evaluation |

### 4. Company News (15%)

| Score | Indicators |
|-------|------------|
| 9-10 | Fresh funding, new CTO/VP Eng, major expansion |
| 7-8 | Product launches, tech team growth |
| 5-6 | Steady growth, occasional hiring |
| 3-4 | Limited recent news |
| 1-2 | Layoffs, budget cuts, hiring freeze |

### 5. Reachable Contacts (10%)

| Score | Indicators |
|-------|------------|
| 9-10 | Multiple contacts identified, warm intro possible |
| 7-8 | Key personas found, standard outreach viable |
| 5-6 | Some contacts identifiable |
| 3-4 | Limited public presence |
| 1-2 | No identifiable contacts |

**Target Personas:** VP/Director Engineering, Platform Lead, DevEx Lead, DevOps Manager, Engineering Manager, Security Lead
